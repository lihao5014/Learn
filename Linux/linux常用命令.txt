7.使用git统计代码行数命令：
（1）git log --author="username" start_commit_id..end_commit_id --pretty=tformat: --numstat | gawk '{ add += $1 ; subs += $2 ; loc += $1 - $2 } END { printf "added lines: %s removed lines : %s total lines: %s\n",add,subs,loc }'
（2）将'start_commit_id'改为第一次提交的id，'end_commit_id'改为最后一次提交的id，'username'改为git用户名。就可以统计出两次提交间代码的变化行数。

33.命令行中使用git提交代码流程：
（1）git pull                         //同步远程仓库代码，以避免代码提交上库后造成冲突。
（2）git status                       //查看当前本地仓库状态。
（3）git add *.h *.c main.cpp         //提交代码到本地git缓冲区。
（4）git commit -m "提交代码修改"     //推送代码到本地git仓库。
（5）git push                         //提交本地代码到远程仓库。
（6）git log                          //查看代码提交日志，保证代码已经提交成功。

34.git提交大于100M的文件：
（1）git lfs install                                                      #git bash命令行中切换到本地仓库目录，开启LFS功能
（2）git lfs track "*.tar" "*.tar.bz2" "*.tar.xz" "*.rar" "*.tar.gz"      #设置LFS要管理的文件类型
（3）git add .gitattributes                                               #配置远程仓库。执行完上面的命令后，会生成一个.gitattributes文件，要将其上传到远程gitee仓库。。
（4）git commit -m "提交.gitattributes文件"                               #这里把.gitattributes和大文件分开上传
（5）git push origin master                                               #如果提交不了，后面可以加一个-f
（6）git add linux-5.4.212.tar.xz                                         #上传大文件。也可以使用TortoiseGit提交大文件。
（7）git commit -m "提交linux-5.4.212源码"
（8）git push origin master -f                                            #如果报“batch response: LFS only supported repository in paid enterprise”错，可以删除./git/hooks/pre-push文件再试
